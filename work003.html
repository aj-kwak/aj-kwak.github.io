Configuration

I built the bot as a Python flask app earlier and hosted it on Heroku. However, in this thread we shall go with a more beginner-friendly approach. We will use a third-party platform to add some extra functionality to the bot and while hosted in the cloud; the bot won't go offline whether your pc is powered on or off.

Bot Setup

Once created an account with TradingView, Wunderbit, and Binance. The procedure starts with adding Binance API keys to the Wunderbit Account. While signing up to Wunderbit for the first time, you will get prompted to add the Binance exchange API key. Here is: How to create Binance API -tutorial. Remember to set desired permissions on the API "Edit" -panel; you want to "Enable Futures" if using Binance Futures account to trade.

Next up, Go to "Bots" -page on Wunderbit and click on "Add New" -button. It's where to set extra logic to your bot such as, Stop Loss, Take Profit, etc. On the right side you will see the guides on how to automate Strategy and Study -scripts. The script we use is Strategy; Therefore, use this tutorial to set up the automation to the bot.

As this script swings from long to short and vice versa; i.e. being in a trade non-stop. It is only needed to set the strategy.entry long, and strategy.entry short -comments to the script. That's about it; if followed the steps, you should have a working trading bot hosted in the cloud by now. There is a responsive support chat on Wunderbit app in a case of technical issues.

Note; there are thousands of community -scripts available on TradingView. If this script doesn't suit you, pick another one. Needless to say, learning the procedure with one script increases the likelihood of successfully setting up another script with slightly variant configuration methods.
