002 : Deploying A BEP-20 Token On BSC

The following token contract is set available for download at the end of this thread. It contains functions such as; Auto-liquitity Mechanism, Marketing Tax, Bonus Tokens, Anti-rug, and Anti-whale mechanisms. By taking the steps in this thread, you can get a BEP-20 token contract including interesting tokenomics deployed to the BSC.

This token contract development was inspired by a mini YouTube video -series on "How to Make a BEP-20 Token with Tokenomics and deploy it on the BCS". While reading the comments, I noticed a lot of viewers demanding advice on; how to change the marketing fee to BNB instead of receiving this fee in the form of native token.

This request makes sense, as who would want to dump native tokens on project supporters, right? The main difference between this smart contract versus the gcoach token contract; is the way fees are handled and, *IMPORTANT* in this contract marketing fee is received in BNB instead of "yourOwnToken".

In addition to keeping the tokenomics safe and more desirable for investors. We have added some "No-rug and Anti-whale" -functions; A maxbuywallet, capped total tax%, and no-ability to mint more tokens. As well as, bonus tokens are re-distributed to all current holders on each transaction.

About The Smart Contract

To be quite honest, this contract reminds me of the Safemoon contract in a way of the auto-liquidity function and bonus tokens. However, after the current additions, this contract is fair for all parties and seems way more desirable in the eyes of an investor.

Obviously you will need a project with the potential to back up long term sustainability. Tokenomics comes first; Bad tokenomics can break a good project, yet good tokenomics can carry a bad project. Safemoon being a great example of this - By reaching a fully diluted market cap of $550,059,263.50 on the peak of its ATH.

If you ask me, a fair launch would be the way to go, no reserved tokens. The marketing tax will fund the project. If set to 2%, you will receive 2% of all transactions. In practice; the token does 100k USD in daily trading volume; received tokens in BNB ~2k USD, distributed to the external marketing wallet.


SMART CONTRACT FUNCTIONS:
  1. Auto-generating liquidity mechanism - A percentage of all transactions is split and transferred to the PCS liquidity pool.
  2. Bonus Tokens - A percentage of all transactions is re-distributed to current token holders as bonus tokens.
  3. Marketing Fee - A percentage of all transactions is distributed to your external wallet as BNB.
  4. Anti-whale -function Maxbuywallet - won't allow wallets to buy more than a certain percentage of the total supply.
  5. A No-rug -function - won't allow owner to mint more tokens after deployment; this way owner can't create tokens out of thin air and dump them on rest of the holders.

Functions limiting owner privileges are as, if not more important as the functions benefiting the investor. Limiting bad actors such as whales from entering the network. The contract has its limitations for potential bad actors on both sides of the contract.

Editing the Tokenomics

You can safely edit all lines in the contract marked with //Comments. Copypaste the code from the txt -file downloaded at the end of this thread, create a new solidity contract file in the Remix Ethereum IDE, and paste it into the editor.


Edit the following lines:
  • LINE: 412 - Change the contract name (it's not the token name or ticker, just a name for the contract set "Testing" as default).
  • LINE: 416 - Change Marketing address to any of your wallet address.
  • LINE: 429 - Change total supply here.
  • LINE: 433 - This is the token name.
  • LINE: 434 - This is the token ticker.
  • LINE: 438 - This is the fee to change the bonus tokens percentage... Set to 4% by default
  • LINE: 441 - This is the complete liquidity fee.. Set to 8 by default, what happens under the hood is this splits in two. 4% to marketing wallet and 4% to the liquidity pool.
  • LINE: 444 - This is the marketing fee. It is being extracted from the total liquidity fee. If you set it to 7 then the liquidity fee would be 1% and marketing would get 7% out of the entire 8% liquidity fee.
  • That's all you need to edit. Rest of the lines marked with //Comments are pretty self-explanatory, and mostly referring to interacting with the contract on BSC after deployment.


Deploying the Contract

Once set the desired tokenomics, it's time to compile the contract with the Remix Solidity Compiler. A few things to keep in mind; always use the correct contract/address. Make sure you are logged in to the correct Metamask. Account address must match with the one set in the contract. Also remember to use the same compiler/solidity version (0.8.13) in all steps of the process (for example while verifying the contract address on BSCscan).


Setting for Remix Solidity Compiler:
  • Set compiler version to 0.8.13.
  • Set language to Solidity.
  • Enable Optimization and set value to 200.
  • Open the drop-down menu and choose the "Testing" -Contract: (TestToken.sol).
  • Move to the 4th tab: "Deploy & Run Transactions".
  • Select Environment "Injected Web3"Make sure your Metamask or other Web3 wallet is open and logged in a correct account.
  • Open the drop-down menu and choose the "Testing" -Contract again: (TestToken.sol).
  • Once you are ready with all previously mentioned settings, you can go and deploy your token by clicking the "Deploy" -Button.
